# IMPLEMENTATION PLAN

Project: Lightweight Physics-Informed Neural Network for Automated THz Parameter Extraction in Graphene-Based Nanosystems

This is the working implementation document for the full research build. It combines the earlier engineering roadmap with the requested week-by-week research plan and adds practical improvements needed to make the project reproducible, testable, and easy to execute with two collaborators.

## 1. Project Goal

Develop a lightweight Python/PyTorch framework that accepts THz spectroscopy signals and estimates the hidden transport parameters:

- `tau_intra`: intraband carrier relaxation time
- `tau_inter`: interband recombination lifetime
- `k_transfer`: charge transfer coefficient

The framework should:

1. Learn from synthetic THz curves generated by a simplified physics model.
2. Ingest literature-derived curves digitized from published papers.
3. Train baseline and lightweight neural architectures.
4. Add physics consistency through a PINN-style loss.
5. Support generalization analysis across material systems.
6. Produce calibrated uncertainty estimates.
7. Run on standard laptop hardware without unnecessary complexity.

## 6. Dataset Design

### 6.1 Synthetic dataset format

Use one metadata table plus one curve store.

Required columns:

- `sample_id`
- `t`
- `signal`
- `tau_intra`
- `tau_inter`
- `k_transfer`
- `A1`
- `A2`
- `noise_std`
- `structure_type`
- `split`
- `source`

Recommended storage:

1. Save raw generated curves for traceability.
2. Save processed fixed-length tensors separately.
3. Keep dataset versioning information in a manifest file.

### 6.2 Literature dataset format

Required columns:

- `sample_id`
- `paper_id`
- `paper_title`
- `figure_id`
- `curve_label`
- `material_system`
- `t_or_x_axis`
- `signal`
- `digitization_method`
- `normalization_method`
- `source_url`
- `notes`

If original labels are incomplete, keep raw extracted points and cleaned/resampled points separately.

### 6.3 Processed dataset format

Store ready-to-train arrays in a fixed schema:

- input tensor shape: `[num_samples, num_points]` or `[num_samples, 1, num_points]`
- target tensor shape: `[num_samples, 3]`
- optional metadata file for sample provenance and split assignment

## 7. Preprocessing Pipeline

The preprocessing path should be shared as much as possible between synthetic and literature data.

Pipeline:

1. Load raw curve and metadata.
2. Remove invalid points and duplicates.
3. Align all curves to a common time grid.
4. Interpolate missing samples if needed.
5. Apply mild smoothing only when justified.
6. Normalize amplitude using a clearly defined policy.
7. Resample to a fixed-length vector.
8. Split by sample, not by point, to avoid leakage.
9. Save processed tensors and labels.

Important implementation rule:

- Keep preprocessing parameters in config files so the exact pipeline can be rerun later.

## 9. Model Architecture Roadmap

### 9.1 Baseline MLP

Use this first because it is the fastest inverse-modeling baseline.

Suggested structure:

- input: 1000-point normalized THz vector
- hidden layers: 512 -> 256 -> 128
- activation: ReLU or GELU
- optional dropout
- output: 3 regression values

Purpose:

- verify the entire data pipeline
- establish a baseline metric set
- detect preprocessing or label issues early

### 9.2 Lightweight 1D CNN

Use this second to exploit local waveform patterns.

Suggested structure:

- 1D convolution blocks
- normalization layer
- small pooling or strided convolution stages
- compact dense regression head
- 3 output parameters

Purpose:

- improve learning of local signal features
- compare against MLP on the same dataset

### 9.3 Physics-Informed Neural Network

Use this after the baseline stack is stable.

Suggested design:

- backbone: MLP or 1D CNN
- parameter head for `tau_intra`, `tau_inter`, `k_transfer`
- reconstruction head or differentiable physics layer
- physics-aware hybrid loss

Loss structure:

- `L_prediction` for direct parameter prediction
- `L_reconstruction` for regenerated THz curve consistency
- `L_constraint` for physics bounds and consistency terms

Total loss:

```text
L_total = L_prediction + lambda_1 * L_reconstruction + lambda_2 * L_constraint
```

## 10. Training Pipeline

Training should be config-driven and reproducible.

Flow:

1. Load experiment config.
2. Set random seeds.
3. Load processed train, validation, and test sets.
4. Build the chosen model.
5. Define loss, optimizer, and scheduler.
6. Train with validation after every epoch.
7. Save best checkpoint.
8. Run test evaluation.
9. Export metrics, figures, and predictions.

Required features:

- early stopping
- checkpointing
- gradient clipping
- learning-rate scheduling
- batch-level logging
- reproducibility controls

Recommended optimizer path:

- start with Adam
- add scheduler such as ReduceLROnPlateau or cosine decay

- parity plots

## 12. Uncertainty Estimation

### 12.1 Monte Carlo Dropout

Workflow:

1. Keep dropout active at inference.
2. Run many stochastic forward passes.
3. Compute mean and standard deviation.
4. Produce prediction intervals.

### 12.2 Ensemble Models

Workflow:

1. Train multiple models with different random seeds.
2. Aggregate predictions.
3. Measure spread and uncertainty.
4. Compare against Monte Carlo Dropout.

Recommended outputs:

- point prediction
- standard deviation
- 95 percent prediction interval
- optional calibration plot

Objective:

Create a robust training dataset.

Tasks:

```text
S(t) = A1 * exp(-t / tau_intra) + A2 * exp(-t / tau_inter) + k_transfer * f(t)
```

- sample parameter ranges:
  - `tau_intra`: 100 to 500 fs
  - `tau_inter`: 1 to 10 ps

## 14. Suggested Improvements

These additions will make the project more reliable and easier to defend scientifically.

1. Add a formal configuration system.
	- Use YAML or TOML files for dataset, training, and experiment settings.
	- This prevents hidden hard-coded assumptions.

2. Add dataset manifests and versioning.
	- Save a manifest for every generated dataset batch.
	- Record parameter ranges, noise settings, and preprocessing parameters.

3. Separate raw, processed, and evaluation data.
	- Do not overwrite raw curves.
	- Keep processed tensors immutable once published for an experiment.

4. Add test coverage early.
	- Unit test the synthetic generator, preprocessing, and shape checks.
	- This will catch most pipeline errors before training starts.

5. Add baseline ablations.
	- Compare training with and without noise.
	- Compare fixed-length resampling choices.
	- Compare MLP, CNN, and PINN on the same split.

6. Add calibration analysis.
	- Uncertainty estimates should be evaluated, not just produced.
	- Track whether the reported intervals match actual errors.

7. Add a domain-shift experiment.
	- Synthetic-to-literature transfer is a core scientific risk.
	- Make that gap explicit in the report.

8. Add parameter identifiability checks.
	- `tau_intra`, `tau_inter`, and `k_transfer` may be coupled.
	- Analyze correlations to understand whether the inverse problem is well posed.

9. Add training logs and experiment summaries.
	- Save metrics as CSV or JSON.
	- Save plots with consistent filenames and timestamps.

10. Add a simple inference API.
	 - A notebook is good for research.
	 - A small script or app is better for repeated demos.
